## 🧭 今回作るアプリ：『おみくじ習慣トラッカー』
ボタンを押すと今日のみくじ（大吉・吉など）が引けて、引いた結果（履歴）が下にリストでどんどん蓄積されていく、1画面の簡単なアプリです。
### なぜこれが例題に最適なのか？
 * **UI（ボタン、リスト）の作成**
 * **データ構造（おみくじの型、履歴の配列）の管理**
 * **ロジック（ランダムに結果を選ぶ）の実装**
   この3つの要素が含まれているため、aiderに指示を出す練習として満点の内容です。

## 🛠️ CLIのみで完結する事前準備（5つのコマンド）
まずはターミナルを開き、以下のコマンドを順番に実行してプロジェクトの土台を立ち上げます。
```bash
# 1. 練習用のフォルダを作って移動
mkdir OmikujiCLI
cd OmikujiCLI

# 2. Swiftのプロジェクト（コマンドラインツール）を自動生成
swift package init --type executable

# 3. Gitを初期化して最初の状態を保存
git init
git add .
git commit -m "initial commit"

```
これで、CLIで動くSwiftプログラムの骨組みが完成しました。
## 🚀 aiderを起動する
プロジェクトのフォルダ（OmikujiCLI）にいる状態で、自動コンパイル（swift build）をセットしてaiderを起動します。
```bash
aider --test-cmd "swift build" --auto-test

```
起動したら、aiderのチャット欄に以下の指示を1ステップずつ出していきましょう。aiderが編集すべきファイル（Sources/main.swift または Sources/OmikujiCLI/main.swift ※Swiftのバージョンにより自動生成される位置）を自分で見つけて書き換えてくれます。
## 🤖 aiderへの「CLI版・3ステップ指示書」
### STEP 1：データ構造とデータの保存場所を作る
まずは、おみくじのデータ型と、履歴を保存する配列を作らせます。
> **指示1:**
> おみくじの結果を管理する構造体 Omikuji を定義してください。
> プロパティには、unsei（文字列：「大吉」「吉」「中吉」「凶」）、date（文字列：引いた日時）を持たせてください。
> また、これまでに引いたおみくじの履歴を保存するための配列 var history: [Omikuji] = [] を定義してください。
> 
（※自動で swift build が走り、コンパイルが通ることを確認してください）
### STEP 2：おみくじを引くロジックと、履歴表示を作る
次に、ランダムでおみくじを引く関数と、現在の履歴をターミナルに綺麗に表示する関数を作らせます。
> **指示2:**
> 以下の2つの関数を実装してください。
>  1. drawOmikuji(): 大吉、吉、中吉、凶の中からランダムに1つを選び、現在の日時（yyyy/MM/dd HH:mm:ss形式）とともに Omikuji 構造体を作成し、配列 history の先頭（0番目）に追加する関数。
>  2. showHistory(): 現在の履歴の配列をループで回し、ターミナルに「[日時] あなたの運勢は: 大吉」のような形で一覧表示する関数。履歴が空の場合は「まだおみくじを引いていません」と表示してください。
> 
### STEP 3：ターミナルで対話できるメインメニューを作る
最後に、ユーザーがキーボード入力で操作できる対話型のループ（メニュー画面）を作らせます。
> ## **指示3:**
> プログラムが起動したら、ユーザーが終了するまで繰り返す対話型のメインメニューを実装してください。
> ターミナルに以下を表示し、ユーザーの入力を待つようにします。
> ## 1: おみくじを引く
> 2: 履歴を見る
> 3: 終了
> 1が押されたらおみくじを引いて結果を表示、2が押されたら履歴一覧を表示、3が押されたらプログラムを終了するようにしてください。
> 
## 🏃‍♂️ 完成したアプリをCLIで実行する
aiderがすべてのコンパイルを通し、Gitにコミットしてくれたら、aiderを一度終了する（または別タブのターミナルを開く）か、aider内で /run を使ってアプリを動かしてみましょう！
**実行コマンド：**
```bash
swift run

```
画面にメニューが表示され、1 や 2 を入力してエンターを押すことで、自分で作ったおみくじアプリがターミナル上でガシガシ動くはずです。
## 🤝 困ったときのCLIサポート
もし途中で「ビルドエラー（Build Failed）」で止まったら、以下のプロンプトをaiderに投げてください。
 * **エラーを丸投げして直させるとき：**
   > 「ビルドエラーが発生しています。エラーログの内容を解析して、main.swift の文法を正しく修正してください」
   > 
 * **コードを全部リセットしてやり直したいとき（Gitの力）：**
   aiderを終了し、ターミナルで git reset --hard HEAD と打てば、1つ前の指示が成功した時点（または初期状態）まで一瞬で巻き戻せます。
完全CLIでのアプリ開発、めちゃくちゃシブくてカッコいいです。まずは最初のコマンド（swift package init）から試してみてください！結果の報告を待っています！
