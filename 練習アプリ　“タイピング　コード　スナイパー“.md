## 🗺️ 完全CLI開発：3ステップ・ロードマップ
大まかな流れは以下の通りです。
 1. **【Mac CLI】**：main.swift を作成し、文字コードのロジックとログ出力を完璧にする。
 2. **【iPhoneシミュレータ】**：コードをSwiftUIに移植し、Macのコマンドでシミュレータを起動してテストする。

### 🛠️ ステップ1：Macアプリ（コマンドライン）でロジックを磨く
まずは一番素早くコンパイルと実行ができる「Macのターミナル専用ツール」としてロジックの土台を作ります。
 1. **フォルダの作成とaiderの起動**
   ```bash
   mkdir StringSniper
   cd StringSniper
   aider main.swift
   
   ```
 2. **aiderへの指示**

   ```bash
   swiftc main.swift -o StringSniper
   ./StringSniper
   
   ```
   ここで、あえて「NFC/NFDのバグ」を発生させ、ターミナルに出たHexログをそのままaiderにコピペして、**「自動で文字コードを綺麗にする堅牢なロジック」**へとaider自身に修正させます。
### 💻 ステップ2：iPhoneシミュレータで動かす
Mac上でロジック（中身）が完璧になったら、それをiPhoneアプリ（外見・SwiftUI）の形に移植し、Macの画面上のシミュレータで動かします。
 1. **SwiftUIプロジェクトの準備**
   aiderに指示して、最小構成のiOSプロジェクトのファイル群（App.swift, ContentView.swift, .xcodeproj）をコマンドライン上で生成させます。
 2. **ターミナルからシミュレータへビルド＆起動**
   Xcodeの再生ボタンを押す代わりに、ターミナルで以下のコマンドを実行します。
   ```bash
   # 1. 起動中のシミュレータのID（UUID）を確認
   xcrun simctl list devices boot
   
   # 2. ターミナルからシミュレータ向けにビルド
   xcodebuild -project StringSniper.xcodeproj -scheme StringSniper -sdk iphonesimulator -configuration Debug build
   
   # 3. シミュレータにアプリをインストールして強制起動
   xcrun simctl install booted build/Build/Products/Debug-iphonesimulator/StringSniper.app
   xcrun simctl launch booted com.yourname.StringSniper
   
   ```
 3. **シミュレータのログをターミナルで監視する**
   アプリがシミュレータ内で吐き出す print ログ（Hexダンプなど）は、以下のコマンドで**Macのターミナル上にリアルタイムで流す**ことができます。
   ```bash
   xcrun simctl spawn booted log stream --predicate 'process == "StringSniper"'
   
   ```

### 1. 一番おすすめ：GitHubの「RAW」URL
今回の開発でも使ったGitHubには、ファイルの中身のテキストだけをそのまま出力する「Raw」という機能があります。
 * **具体的なURLの例：**
   https://raw.githubusercontent.com/run-llama/llama_index/main/README.md
 * **仕組み：**
   GitHubのリポジトリにある README.md などのファイルをブラウザで開き、画面右上にある **「Raw」ボタン** を押すと、このURLに切り替わります。
 * **練習になる理由：**
   GitHub上のテキストはサーバー（Linux環境）に保存されているため、日本語が含まれていると**高確率で「NFD（濁点分離形式）」の文字コード**になっています。これを自分のMacやiPhoneから普通にタイピングすると、見た目は同じなのに文字コードがズレて「❌不一致」になり、今回の文字コード問題の最高のデバッグ練習になります。
### 2. 手軽に試せる：青空文庫（テキスト版）のURL
ネット上で公開されている著作権の切れた小説などのテキストデータを直接読み込ませるのも楽しいです。
 * **具体的なURLの例：**
   https://www.aozora.gr.jp/cards/000035/files/1567_ruby_4945.zip （※アプリ側でZIP解凍が必要になるため、最初は解凍済みのプレーンテキストURLが理想です）
 * **仕組み：**
   「〇〇.txt」で終わるような、インターネット上の公開テキストファイルのURLをそのまま指定します。
 * **練習になる理由：**
   長文をタイピングする練習になるほか、古いファイルだと「Shift_JIS」などの異なる文字コードで保存されている場合があり、アプリ側で「UTF-8への変換」や「文字化け対策」の堅牢性をテストする良い教材になります。
### 3. 自作のテストデータ：Gist（ギスト）のURL
自分で「わざと罠を仕込んだ日本語テキスト」を作って検証したい場合は、GitHubの **Gist（ギスト）** という簡易コード共有サービスを使うのが一番簡単です。
 1. GitHub Gist にアクセスする。
 2. ファイル名（例: test.txt）を入れ、本文に「Aiderとollamaでコーディング」など、検証したい日本語を入力する。
 3. 「Create secret gist」または「Create public gist」で保存する。
 4. 画面右上にある **「Raw」ボタン** を押す。
 5. 表示されたページのURL（https://gist.githubusercontent.com/.../raw/.../test.txt）をアプリに入力します。
